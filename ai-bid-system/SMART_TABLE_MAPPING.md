# 🎯 智能表格字段映射功能说明

## 📋 功能概述

系统现在能够**精准识别复杂简历表中每个空白单元格应该填什么内容**，通过智能字段映射引擎自动将人员库数据填充到正确的位置。

---

## ✨ 核心功能

### 1. **智能空白单元格识别**
- 自动识别表格中的空白单元格
- 提取每个空白单元格的语义标签（如"姓名"、"学历"、"职称"）
- 支持复杂表格结构（合并单元格、多层表头）

### 2. **自动字段映射**
- 将空白单元格标签自动映射到人员库字段
- 支持多种匹配策略：
  - **精确匹配**：标签完全匹配字段名（置信度：高 ●●●）
  - **模糊匹配**：标签包含字段关键词（置信度：中 ●●○）
  - **未匹配**：无法找到对应字段（置信度：低 ●○○）

### 3. **可视化标记**
- 在原始表格预览中，用**黄色背景**标记空白单元格
- 显示每个空白单元格的：
  - 📝 语义标签（如"姓名"）
  - → 映射字段（如 `name`）
  - ●●● 置信度指示器

### 4. **智能数据填充**
- 从人员库自动提取数据填充到对应的空白单元格
- **只填充空白单元格**，保留表格中的固定内容（如"性别：男"）
- 支持项目经历的多行展开

### 5. **映射详情查看**
- 点击"查看映射详情"按钮查看完整的映射关系
- 显示每个空白单元格的位置、标签、映射字段和置信度
- 提供映射统计信息

## 🔧 技术实现

### 前端核心文件
- **`frontend/src/utils/tableFieldMapper.js`** - 智能字段映射引擎
  - 字段映射规则定义
  - 智能匹配算法
  - 单元格映射构建
  - 数据填充函数

- **`frontend/src/pages/CreateBid.jsx`** - 主界面集成
  - 单元格映射状态管理
  - 可视化标记渲染
  - 智能填充逻辑

### 后端核心文件
- **`backend/routes/merge_docs.py`** - 文档合并优化
  - 智能填充逻辑（只填充有值的单元格）
  - 详细的填充日志输出

## 🎨 用户体验

### 解析阶段
上传标书后，系统会自动：
1. 识别所有动态表格
2. 构建单元格映射
3. 在控制台输出映射统计：
   ```
   📊 [智能映射] 表格 0: 8 个空白单元格, 7 个已映射 (87.5%)
      ✓ [1,0] 姓名 → name (high)
      ✓ [1,2] 学历 → education (high)
      ✓ [1,3] 职称 → title (high)
   ```

### 填写阶段
在动态表格卡片中：
1. **表格标题下方**显示映射统计：
   ```
   🎯 智能识别: 8 个空白单元格  已映射 7  未映射 1  (87.5%)
   ```

2. **原始表格预览**中：
   - 空白单元格显示为黄色背景
   - 显示语义标签和映射字段
   - 显示置信度指示器

3. **选择人员和职位后**：
   - 系统自动根据映射关系填充数据
   - 只填充空白单元格，保留固定内容
   - 支持项目经历的多行展开

4. **查看映射详情**：
   - 点击按钮查看完整映射表
   - 显示每个空白单元格的详细信息

### 导出阶段
后端合并文档时：
1. 只填充有值的单元格
2. 保留原始表格的固定内容
3. 输出详细的填充日志：
   ```
   ✓ [1,0] 姓名: '' → '张三'
   ⏭️ [1,1] 性别: 保留原值 '男'
   ✓ [1,2] 学历: '' → '本科'
   ```

## 🚀 使用流程

### 1. 上传标书
- 上传包含复杂简历表的标书文档

**填充后**：
```
┌────────┬──────┬────────┬────────────┐
│ 姓名   │ 性别 │ 学历   │ 职称       │
├────────┼──────┼────────┼────────────┤
│ 张三   │ 男   │ 本科   │ 高级工程师 │
└────────┴──────┴────────┴────────────┘
```

## 📝 注意事项

1. **映射规则可扩展**：如需支持更多字段，可在 `tableFieldMapper.js` 的 `FIELD_MAPPING_RULES` 中添加

2. **置信度说明**：
   - 高置信度（●●●）：可以直接使用
   - 中置信度（●●○）：建议检查
   - 低置信度（●○○）：需要手动调整

3. **项目经历处理**：
   - 如果人员有多个项目经历，系统会为每个项目生成一行
   - 项目经历字段会自动拼接（用"；"分隔）

4. **未映射字段**：
   - 如果某些空白单元格未映射，用户可以在填充后手动编辑
   - 后续可以在人员库中补充信息后重新填充
