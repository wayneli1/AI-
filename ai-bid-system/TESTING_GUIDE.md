## 🐛 问题排查

### 问题1：HTML长度=0

**原因：** `tableHtml` 没有正确传递

**排查步骤：**
1. 检查后端 `response.py` 是否添加了 `tableHtml` 字段
2. 检查后端 `parse_bid.py` 第64行和第77行是否传递了 `tableHtml`
3. 重启后端
4. 清除浏览器缓存，刷新前端

**验证命令：**
```bash
# 检查后端代码
cd C:\Users\Coremail\Desktop\biaoshu\ai-bid-system\backend
grep -n "tableHtml" schemas/response.py
grep -n "tableHtml" routes/parse_bid.py
```

---

### 问题2：Dify 返回格式错误

**症状：** 前端报错 `JSON parse error`

**排查步骤：**
1. 查看后端日志中的 `✅ [Dify] 原始返回`
2. 检查是否包含 markdown 标记（如 ```json）
3. 在 Dify 工作流中添加清洗逻辑（参考 DIFY_AI_TABLE_UNDERSTANDING_GUIDE.md）

---

### 问题3：字段识别错误

**症状：** "姓名"被填充到"职务"列

**排查步骤：**
1. 检查 Dify 工作流的提示词是否正确
2. 检查 `table_html` 是否完整（在 Dify 运行日志中查看）
3. 调整 LLM 的 temperature（建议 0.1-0.3）
4. 在提示词中增加示例

---

### 问题4：填充位置错误

**症状：** 数据追加到表格末尾

**排查步骤：**
1. 检查前端 `CreateBid.jsx` 第3283行的 `findIndex` 逻辑
2. 检查 `dynamicTableEdits` 的数据结构
3. 清除浏览器缓存

---
