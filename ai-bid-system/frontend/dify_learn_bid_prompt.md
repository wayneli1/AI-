# Dify 工作流配置指南 - 投标文件学习

## 工作流结构
```
开始节点 → LLM节点 → 结束节点
```

## 输入变量（开始节点）
| 变量名 | 类型 | 说明 |
|--------|------|------|
| `markdown_content` | text-input | 分块后的投标文件 Markdown 文本 |

## 输出变量（结束节点）
| 变量名 | 来源 | 类型 |
|--------|------|------|
| `text` | LLM节点的 text 输出 | string |

## LLM 提示词（复制到 Dify 的 LLM 节点）

```
你是一个投标文件信息提取引擎。请从以下投标文件片段中提取所有公司相关信息。

规则：
1. 以 "字段名: 值" 的形式提取所有明确出现的公司信息
2. 字段名必须使用中文原文，例如"法定代表人"而不是"legal_rep"
3. 不要编造信息，只提取文本中明确出现的
4. 不需要提取评分标准、技术要求，只关注公司基本信息（如公司名称、地址、法人、联系方式、资质等）
5. 只输出JSON，不要任何解释

输出格式：
{
  "fields": [
    {"key": "公司名称", "value": "xxx有限公司"},
    {"key": "法定代表人", "value": "张三"},
    {"key": "联系电话", "value": "13800138000"}
  ]
}

待提取文本：
{{markdown_content}}
```

## 配置步骤

1. **创建新工作流**
   - 在 Dify 控制台创建新应用
   - 选择"工作流"模式

2. **配置开始节点**
   - 添加一个开始节点
   - 添加输入变量：`markdown_content` (text-input)
   - 复制上面的提示词到 LLM 节点

3. **配置 LLM 节点**
   - 模型选择：qwen3.5-plus 或其他中文模型
   - 温度：0.1（保持稳定输出）
   - 最大 token：65501
   - 粘贴上面的提示词

4. **配置结束节点**
   - 添加结束节点
   - 输出变量：`text` (从 LLM 节点的 text 输出)

5. **获取工作流 ID**
   - 发布工作流
   - 复制工作流 ID（格式：app-xxxxxxxxxxxxx）

6. **更新环境变量**
   - 在 `.env.local` 文件中设置：
     ```
     VITE_DIFY_EXTRACTION_API_KEY=你的Dify API Key
     VITE_DIFY_API_BASE=https://api.dify.ai/v1
     ```

## 测试数据示例

```json
{
  "inputs": {
    "markdown_content": "投标人：xxx有限公司\n法定代表人：张三\n注册资本：500万元\n联系电话：13800138000\n地址：北京市朝阳区"
  }
}
```

预期输出：
```json
{
  "fields": [
    {"key": "投标人", "value": "xxx有限公司"},
    {"key": "法定代表人", "value": "张三"},
    {"key": "注册资本", "value": "500万元"},
    {"key": "联系电话", "value": "13800138000"},
    {"key": "地址", "value": "北京市朝阳区"}
  ]
}
```

## 注意事项

1. **字段名必须是中文**：系统直接使用提取的中文字段名
2. **只提取明确信息**：不要猜测或编造
3. **JSON格式必须正确**：确保输出是有效的 JSON
4. **批量处理**：系统会自动分块处理长文档
5. **跨文档分析**：上传多份文档后，系统会自动对比高频字段
```
