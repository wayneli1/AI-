# 服务手册合并工具使用指南

## 概述

本工具用于将AI投标系统生成的包含服务手册暗码标记的Word文档，替换为实际的服务手册内容。采用"AI输出暗码 + Python合并"的策略，100%保留服务手册的原始格式。

## 系统架构

```
前端系统 (React) → AI填写暗码 → 导出.docx + .json → Python合并脚本 → 最终文档
```

## 文件说明

### 1. 核心脚本
- `merge_manuals_final.py` - **推荐使用**，XML层面操作，100%格式保留
- `merge_manuals_practical.py` - 基础版，保留基本格式
- `merge_manuals_reliable.py` - 使用docxcompose库，专业合并

### 2. 测试脚本
- `test_xml_merge.py` - XML合并测试
- `simple_integration_test.py` - 简单集成测试
- `test_workflow.py` - 完整工作流测试

## 安装依赖

```bash
# 必需依赖
pip install python-docx requests

# 如果使用merge_manuals_final.py（推荐）
pip install lxml

# 如果使用merge_manuals_reliable.py
pip install docxcompose
```

## 使用步骤

### 第一步：从前端系统导出文件

1. 在CreateBid页面选择需要的服务手册
2. AI填写投标书，输出暗码标记如 `[INSERT_DOC:VIP售后手册]`
3. 点击"导出Word"按钮，系统会自动导出两个文件：
   - `已填报_投标书.docx` - 包含暗码标记的文档
   - `mapping_投标书.json` - 暗码到URL的映射文件

### 第二步：运行合并脚本

```bash
# 推荐使用XML层面操作的版本（100%格式保留）
python merge_manuals_final.py "已填报_投标书.docx"

# 或者使用docxcompose版本
python merge_manuals_reliable.py "已填报_投标书.docx"

# 基础版本（简单格式保留）
python merge_manuals_practical.py "已填报_投标书.docx"
```

### 第三步：获取结果

脚本运行成功后，会在同一目录下生成：
- `最终_已填报_投标书.docx` - 合并后的完整文档

## 脚本参数

```bash
# 基本用法
python merge_manuals_final.py <输入docx文件>

# 示例
python merge_manuals_final.py "已填报_项目投标书.docx"
python merge_manuals_final.py "C:\Users\用户名\Desktop\已填报_测试.docx"
```

## 工作流程详解

### 1. AI暗码生成
- AI遇到服务手册相关空白时，输出 `[INSERT_DOC:手册名称]`
- 例如：`[INSERT_DOC:VIP售后手册]`、`[INSERT_DOC:标准售后手册]`

### 2. 前端映射生成
- 前端记录暗码到实际服务手册URL的映射
- 导出为JSON格式：`{"VIP售后手册": "https://supabase.url/vip.docx"}`

### 3. Python合并过程
1. 读取主文档和映射文件
2. 下载服务手册.docx文件
3. 在XML层面进行文档合并
4. 100%保留原始格式（表格、图片、样式等）
5. 生成最终文档

## 技术特点

### ✅ 优势
1. **100%格式保留**：在XML层面操作，不破坏任何原始格式
2. **支持复杂内容**：表格、图片、图表、特殊字体等完全保留
3. **无缝插入**：服务手册内容完美融入主文档
4. **自动化处理**：自动下载、合并、生成最终文件

### ⚠️ 注意事项
1. 确保网络可访问Supabase存储服务
2. 服务手册文件必须是.docx格式
3. 建议使用`merge_manuals_final.py`以获得最佳效果
4. 复杂文档合并可能需要较长时间

## 错误处理

### 常见错误及解决方案

1. **"mapping.json文件不存在"**
   - 确保从前端同时导出了.json文件
   - 检查文件名是否正确：`mapping_投标书.json`

2. **"下载失败"**
   - 检查网络连接
   - 验证Supabase URL是否有效
   - 检查服务手册文件是否存在

3. **"未找到暗码标记"**
   - 确认AI正确输出了暗码标记
   - 检查暗码格式：`[INSERT_DOC:手册名称]`

4. **"依赖库缺失"**
   - 运行：`pip install python-docx requests lxml`
